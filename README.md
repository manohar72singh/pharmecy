# Pharmecy – Online Pharmacy Application

This repository holds a full‑stack online pharmacy project consisting of a React/Vite frontend and an Express backend backed by a SQL database. The application allows customers to browse medicines, manage carts, place orders, upload prescriptions, and more, while administrators can control inventory, users, orders, coupons, and other business operations via an admin dashboard.

---

## 📁 Project Layout

- **backend/** – Node.js/Express server with routes, controllers, middleware, utilities, and SQL models (e.g. using Sequelize or another ORM). Handles authentication, business logic, and file uploads.
- **frontend/** – Vite-powered React app using Tailwind CSS. Contains components, pages, contexts, hooks, and services for interacting with backend APIs.

## 🚀 Key Features

- User authentication with JWT and role-based access control
- Medicine catalog supporting filtering, search, and pagination
- Shopping cart and checkout process for authenticated users
- Prescription upload functionality for regulated products
- Order management with timeline/status tracking
- Admin dashboard to manage medicines, users, orders, coupons, subscriptions, suppliers, etc.
- Payment integration and coupon discount handling
- Stock/ inventory management and scheduled cron tasks

## 🛠 Technology Stack

| Layer     | Technologies                                                                |

## 📦 Setup Instructions

### Backend

```bash
cd backend
npm install
npm run dev    # starts the server on configured port (e.g., 5000)
```

### Frontend

```bash
cd frontend
npm install
npm run dev    # starts the Vite dev server (e.g., http://localhost:5173)
```

The frontend proxies API requests to the backend, so run both concurrently during development.

## 📝 Additional Notes

- Environment variables are managed via `.env` files in each package.
- Uploaded prescriptions are stored in `backend/uploads/prescriptions`.
- Cron jobs are defined in `backend/src/utils/cronJobs.js`.
